### 📮 제출 방식 (2가지 모두 필수!)

- **예측 CSV** → Streamlit 웹앱 업로드
  - MCC 자동 채점 + 리더보드 반영
  - 여러 번 제출 가능 (시간당 10회 · 일일 30회 제한)
- **분석 노트북 (.ipynb)** → LMS (사이버캠퍼스) 별도 제출
  - 코드 검토 + **퀴즈 출제 기준**
  - ⚠️ 출력물(output)이 반드시 남아 있어야 함

> 🚨 **리더보드 점수만으로 최종 성적이 결정되지 않습니다.** 노트북 검토 + 퀴즈 결과가 함께 반영됩니다.

> ⚠️ **동일한 예측 CSV를 제출한 학생이 2명 이상 발견될 경우, 해당 학생 전원을 대상으로 면담을 진행합니다.**

## 6. 🤖 모델 학습 및 저장

> 💡 어떤 알고리즘을 쓸지, 하이퍼파라미터를 어떻게 조합할지는 **여러분의 선택**입니다. 아래는 공통 규칙과 저장/로드 방법만 안내합니다.

### ⚠️ random_state=42 고정 (필수!)

> 🚨 **`random_state` 파라미터가 있는 모델과 데이터 분할에서 `random_state=42`를 고정하세요.**
> (KNN 등 `random_state`가 없는 알고리즘은 해당 없음)

### 🚨 라벨은 문자열 그대로 사용 (중요!)

- `train['label']` (`'POSITIVE'`/`'NEGATIVE'` 문자열)을 그대로 `y`로 사용
- sklearn 분류기는 문자열 라벨을 직접 처리 → `model.predict()`가 문자열 반환
- `LabelEncoder`, 수동 `{POSITIVE: 1}` 매핑은 역매핑 실수 위험 → **비권장**

### 💾 모델 저장 및 불러오기 (Pipeline 권장)

> 🚨 **`final_pipeline.pkl`은 필수 제출 항목입니다.** 교수님이 모델을 로드하여 예측을 재현합니다. 재현 불가 시 감점 대상입니다.

#### 📝 파일 네이밍 규칙

실험별로 모델을 저장하면 나중에 비교하기 편합니다:

```
{알고리즘}_v{버전}_mcc{점수}.pkl
```

| 예시                    | 설명                                                   |
| ----------------------- | ------------------------------------------------------ |
| `logreg_v1_mcc0823.pkl` | LogisticRegression 첫 번째 실험, val MCC 0.823         |
| `svm_v2_mcc0847.pkl`    | SVM 두 번째 실험, val MCC 0.847                        |
| `rf_v1_mcc0831.pkl`     | RandomForest 첫 번째 실험, val MCC 0.831               |
| `final_pipeline.pkl`    | **최종 선택 모델** (가장 좋은 모델을 이 이름으로 복사) |

#### 💾 저장 및 로드

```python
import joblib
```

### 📦 LMS 제출물 목록

## 9. 📏 평가 기준

### 🏆 순위 기준: MCC

$$\mathrm{MCC} = \frac{TP \cdot TN - FP \cdot FN}{\sqrt{(TP + FP)(TP + FN)(TN + FP)(TN + FN)}}$$

- 범위: **[-1, 1]**
- **1** = 완벽한 분류 · **0** = 랜덤 예측 · **-1** = 완전 반대
- MCC가 **음수**일 수 있음 — 랜덤보다 못한 모델이라는 뜻
- Accuracy는 단순 정답률이므로 예측 편향을 반영하지 못합니다. MCC는 TP·TN·FP·FN을 모두 고려하여 **예측의 질**을 더 엄격하게 평가합니다.

### 📊 평가 구조

여러분이 제출하는 CSV는 테스트 데이터 전체(49,997행)에 대한 예측입니다.
제출 즉시 전체 데이터에 대해 MCC를 계산하여 리더보드에 반영합니다.

> ⚠️ **리더보드 점수 ≠ 최종 성적 점수**
>
> 리더보드에 표시되는 MCC는 테스트 데이터 **전체**에 대한 점수입니다.
> 대회 종료 후, 교수님이 테스트 데이터를 두 그룹으로 나누어 각각 MCC를 계산하고 **Min(Group1, Group2)**으로 최종 점수를 결정합니다.
> 따라서 **리더보드 순위와 최종 순위가 다를 수 있습니다.**

- 특정 패턴에 과적합하면 한쪽 그룹의 MCC가 낮아져 최종 점수가 불리해집니다.
- 과도한 반복 튜닝보다 **일반화 성능**에 집중하세요.

### 📐 참고 지표

- **Accuracy**: 전체 예측 중 맞은 비율
- **Precision**: Positive로 예측한 것 중 실제 Positive인 비율
- **Recall**: 실제 Positive 중 Positive로 잡아낸 비율

### 📊 화면 구성

웹사이트는 **세 개의 탭**으로 구성됩니다:

- **[📤 제출] 탭**: CSV 업로드 → 채점 → 결과 요약
- **[📊 분석 · 이력] 탭**: 상세 분석 차트 (레이더, 혼동 행렬, 예측 분포) + 제출 이력 선택
- **[🏆 리더보드] 탭**: MCC 기준 순위 테이블, 본인 행에 **▶** 표시

### 📈 분석 차트 읽는 법

- 🎯 **Confusion Matrix**: 대각선(TP, TN)이 많을수록 ✅, 반대각선(FP, FN)이 적을수록 ✅
- 📊 **예측 분포**: 내 예측(파란색)과 정답(회색) 분포가 비슷하면 균형 잡힌 예측
- 🕸️ **레이더**: 바깥쪽일수록 ✅, 0.5 미만은 랜덤보다 못한 구간

### 🏅 동점 처리

- MCC가 동일한 경우 **동순위**로 처리됩니다 (제출 시각과 무관).

### ⚠️ 동일 CSV 감지

- 이전에 제출한 것과 **동일한 파일**을 다시 제출하면 안내 메시지가 표시됩니다 (점수는 변하지 않으므로 새 모델로 재시도를 권장).
- 다른 학생과 **동일한 예측 결과**가 감지되면 경고가 표시되며, 제출은 정상 처리되지만 내부적으로 기록됩니다. 동일한 baseline을 사용하면 자연스럽게 발생할 수 있으므로, 본인만의 모델 개선을 통해 차별화하세요.
